JavaScript

Developed by Brendan Eich in 1995 while working as an engineer at Netscape. Originally called LiveScript, a poor marketing decision to capitalise on the success of Sun Microsystem's Java language has led to confusion over the two being related ever since.

Javascript was first released with Netscape 2 in early 1996. The same year Netscape submitted JavaScript to the ECMA International and ECMAScript standard was subsequently published later that year.

The JavaScript language has no concept of input or output. It is designed to run as a scripting language in a host environment. The most common host environment is the browser, but JavaScript interpreters can also be found in a huge list of other places, including Adobe Acrobat, Adobe Photoshop, SVG images, Yahoo's Widget engine, server-side environments such as Node.js, NoSQL databases like the open source Apache CouchDB, embedded computers, complete desktop environments like GNOME (one of the most popular GUIs for GNU/Linux operating systems), and others.

Types

JavaScript programs manipulate values, and those values all belong to a type. JavaScript's types are:

  • Number
  • String
  • Boolean
  • Symbol
  • Object
    • Function
    • Array
    • Date
    • RegExp
  • Null
  • undefined
Arrays are a special kind of object. Date and RegExp, are objects that you get for free. And to be technically accurate, functions are just a special type of object. There are also some built-in Error types.

Number

Numbers in JavaScript are, according to the spec, "double-precision 64-bit format IEEE 754 values". This means there is no such thing as integers in JavaScript, resulting in some interesting consequences, so be careful when performing arithmetic.

Standard arithmetic operators are supported: addition, subtraction, multiplication, division, modular arithmetic, etc.

There is a built in object called Math, that provides advanced mathematical functions and constants.

Math.sin(30);
var circumference = 2 * Math.PI * radius;

You can use the parseInt() function, or the unary + operator to convert a string to a number. Similarly, you can parse floating point numbers with the parseFloat() function.

JavaScript has the special values NaN, Infinity and -Infinity.

1 / 0; //Infinity

String

Strings in JavaScript are sequences of Unicode characters. More specifically, strings are sequences of UTF-16 code units; each code unit is respresented by a 16-bit number. Each character is represented by 1 or 2 code units.

Boolean

JavaScript has a bollean type, with possible values true and false (both of which are keywords. All values can be converted to boolean under the following set of rules

  • false, 0, empty strings, NaN, Null and undefined are all converted to false
  • All other values are converted to true
You can perform this conversion explicitly using the Boolean() function.

Boolean(0); //false
Boolean('123'); //true

This conversion is not generally necessary. JavaScript will silently perform this conversion whenever required, when resolving if statements for instance.

The generic bollean operators 'and', 'or' and 'not' are all valid in JavaScript.

NaN

isNaN()

You can test for NaN using the isNaN() function.

isNaN(NaN); //true

Note: isNaN() will also resolve undefined as true. To truely determine if something is NaN, simply compare it itself.

isFinite()

You can test for Infinity, -Infinity and NaN with the built-in function isFinite().

isFinite(0/1); //false
isFinite(-Infinity); //false
isFinite(NaN); //false

parseInt()

You can use the parseInt() function to convert a string to a number. Passing in a number as an optional second argument, allows you to specify which base to use.

parseInt('010',10); //10
parseInt('010',2); //2

A special value NaN (Not a Number) is returned if the string is non-numeric.

parseInt('true',10); //NaN

parseFloat()

Unlike its parseInt() cousin, parseFloat() always uses base 10.

&& (And)

In JavaScript the logical operator && (And) behaves just like one would expect.

&& True False
True True False
False False False

|| (Or)

In JavaScript the logical operator || (Or) behaves just like one would expect.

|| True False
True True True
False True False

! (Not)

In JavaScript the logical operator ! (Not) behaves just as one would expect. When prepended to a variable or statement, the operator converts False to True and vice versa.
